# HOHLOMA — Full-Stack платформа онлайн-записи для студии тату и барбершопа

[![Live Demo](https://img.shields.io/badge/demo-github%20pages-blue)](https://cheech1ko.github.io/HOHLOMA/)
[![Backend API](https://img.shields.io/badge/backend-render-green)](https://hohloma-backend.onrender.com)
[![Playwright Tests](https://img.shields.io/badge/tests-playwright-orange)](https://github.com/Cheech1ko/HOHLOMA/tree/master/e2e-tests)

> Полноценная CRM-система для автоматизации записи клиентов. Проект представляет собой полностью рабочий Full-Stack продукт, готовый к внедрению в реальный бизнес.

## 📋 О проекте

Платформа разработана для студии «Хохлома» (Пенза), объединяющей четыре направления: татуировки, барбершоп, пирсинг и массаж. Система автоматизирует процесс записи клиентов, предоставляет удобную админ-панель для управления заявками и отправляет уведомления на email.

**Ключевые возможности:**
- Онлайн-запись 24/7 с выбором услуги, мастера, даты и времени
- Умный расчёт цены (уровень мастера + наценка за выходные)
- Блокировка занятых слотов (защита от двойных записей)
- Админ-панель с фильтрацией, сортировкой и управлением статусами
- Автоматические email-уведомления клиенту и администратору
- Полное регрессионное тестирование через Playwright

## Бизнес-логика
Наценка за уровень мастера: TOP (+400₽), PROF (+200₽)

Наценка за выходные дни: +200₽ (суббота, воскресенье, праздники)

Блокировка слотов: при записи блокируются 4 слота (текущий + следующие 45 минут)

Статусы заявок: new → confirmed → completed → cancelled

## 🛠 Технический стек

### Frontend
| Технология | Назначение |
|------------|------------|
| **HTML5, CSS3** | Семантическая вёрстка, адаптивный дизайн (Mobile First) |
| **JavaScript (ES6+)** | Вся бизнес-логика, работа с DOM, асинхронные запросы |
| **GSAP + ScrollTrigger** | Плавные анимации при скролле, эффекты появления |
| **Flatpickr** | Календарь с кастомной стилизацией и блокировкой дат |

### Backend
| Технология | Назначение |
|------------|------------|
| **Node.js + Express** | REST API сервер (обработка запросов, маршрутизация) |
| **PostgreSQL** | Реляционная база данных (хранение заявок, мастеров, услуг) |
| **pg** | Драйвер для подключения к PostgreSQL |
| **Nodemailer** | Отправка email-уведомлений |

### DevOps & Инфраструктура
| Технология | Назначение |
|------------|------------|
| **GitHub Pages** | Хостинг фронтенда |
| **Render** | Хостинг бэкенда (Node.js + PostgreSQL) |
| **GitHub Actions** | CI/CD: автоматический запуск тестов при каждом пуше |
| **Playwright** | E2E-тестирование (кросс-браузерное, параллельное выполнение) |
| **Faker.js** | Генерация тестовых данных (рандомизированные сценарии) |

## 🔄 API Эндпоинты

| Метод | Эндпоинт | Назначение |
|-------|----------|------------|
| `GET` | `/api/admin/bookings` | Получение всех заявок (с фильтрацией и сортировкой) |
| `POST` | `/api/bookings` | Создание новой записи |
| `GET` | `/api/bookings/check` | Проверка занятых слотов по мастеру и дате |
| `PATCH` | `/api/admin/bookings/:id` | Обновление статуса заявки |
| `DELETE` | `/api/admin/bookings/:id` | Удаление заявки |

## 🧪 Тестирование

Проект покрыт E2E-тестами с использованием Playwright. Тесты проверяют:

- **Полный цикл записи** — от выбора услуги до отправки формы
- **Блокировку занятых слотов** — защита от двойных записей
- **Кликабельность всех услуг и мастеров** — валидация UI
- **Наценку за выходные дни** — бизнес-логика ценообразования
- **Рандомизированные данные** — генерация уникальных имён, телефонов, email через Faker
